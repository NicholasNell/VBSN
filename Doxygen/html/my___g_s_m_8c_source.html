398  * Sending HTTP requests (GET / HEAD / DELETE)
399  With the command
400  AT#HTTPQRY=<prof_id>,<command>,<resources>,<extra_header_lines> is
401  possible to send a request to HTTP server.
402  Parameters:
403  <prof_id> - Numeric parameter indicating the profile identifier. Range: 0-2
404  <command> - Numeric parameter indicating the command requested to HTTP server:
405  0 GET
406  1 HEAD
407  2 DELETE
408  <resources> - String parameter indicating the HTTP resource (uri), object of the
409  request
410  <extra_header_lines> - String parameter indicating optional HTTP header line.
411  If sending ends successfully, the response is OK; otherwise an error code is
412  reported.
413  Note: the HTTP request header sent with #HTTPQRY always contains the
414  Connection:close line, and it cannot be removed. When the HTTP server answer is
415  received, the following URC is printed on the serial port:
416  #HTTPRING: <prof_id>,<http_status_code>,<content_type>,<data_size>
417  Where:
418  <prof_id> - is defined above
419  <http_status_code> - is the numeric status code, as received from the server
420  (see RFC 2616)
421  <content_type> - is a string reporting the Content-Type header line, as received from
422  the server (see RFC 2616)
423  <data_size> - is the byte amount of data received from the server. If the server doesnt
424  report the "Content-Length:" header line, the parameter is 0.
425  *
426  *
427  *
428  * Sending HTTP POST or PUT
429  With the command
430  AT#HTTPSND=<prof_id>,<command>,<resource>,<data_len>,<post_param>,
431  <extra_header_line> is possible to send data to HTTP server with POST or PUT
432  commands.
433  Parameters:
434  <prof_id> - Numeric parameter indicating the profile identifier. Range: 0-2
435  <command> - Numeric parameter indicating the command requested to HTTP server:
436  0 POST
437  1 PUT
438  <resource> - String parameter indicating the HTTP resource (uri), object of the request
439  <data_len> - Numeric parameter indicating the data length to input in bytes
440  <post_param> - Numeric/string parameter indicating the HTTP Contenttype identifier,
